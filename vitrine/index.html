O ponto de partida

Apps espalhados não são plataforma.

As aplicações internas do CSC nasceram de forma ágil, cada uma resolvendo uma dor real da operação. Mas cresceram dispersas: endereços e portas diferentes, sem dono claro, sem controle de quem acessa o quê e sem trilha de mudanças. Funcionam — mas ainda não formam uma plataforma governável.

Links soltos e endereços decorados

Sem controle de acesso

Sem trilha de auditoria

Cada app numa porta diferente

A solução

Uma vitrine de governança, não só uma lista de links.

Catálogo de apps + permissão por perfil + deploy reprodutível.

Catálogo de apps

Cadastro central com CRUD pela própria tela de Admin.

Launchpad / azulejos

Cards por seção que abrem o app (via URL ou iframe).

RBAC

Perfis agrupam apps; usuários recebem perfis; admin libera tudo.

Boundary de autenticação

Um único authenticate_user(); campo auth_source já pronto para SSO/Entra.

Deploy reprodutível

Docker + Git: o que está na VM é o que está na main.

Auditabilidade

Toggle ativo/inativo em vez de DELETE; atualizado_em carimbado.

Engenharia com critério

Segurança não foi um depois.

CSP + security headers

X-Frame-Options: DENY, entre outros.

Rate-limit no login

5/min por IP, via slowapi.

JWT (3h) + bcrypt

Sessões expiram; senhas nunca em texto plano.

token_version

Reset de senha faz logout global.

Container não-root

Roda como usuário comum (uid 1000).

Modelo de ameaça

Atacante já dentro da rede interna.

HTTPS é o bloqueante de produção (decisão de deploy: TLS + proxy + firewall).
